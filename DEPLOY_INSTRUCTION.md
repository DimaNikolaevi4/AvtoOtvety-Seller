# Инструкция по развёртыванию на BeGet

## 🔒 Безопасная конфигурация деплоя

Этот проект использует шаблоны конфигурационных файлов для безопасного развёртывания на хостинге BeGet. Чувствительные файлы с путями к серверу **не хранятся в репозитории**.

---

## 📁 Файлы, которые не попадают в Git

Следующие файлы добавлены в `.gitignore` и должны создаваться вручную на сервере:

- `.htaccess` — конфигурация Apache/Passenger для BeGet
- `passenger_wsgi.py` — точка входа приложения (может содержать специфичные пути)
- `.env` — переменные окружения (секреты, ключи API, настройки БД)
- `restart.txt` — файл для перезапуска приложения

---

## 🚀 Первое развёртывание (или после сброса)

### Шаг 1: Создайте файл `.htaccess`

1. Скопируйте шаблон:
   ```bash
   cp .htaccess.example .htaccess
   ```

2. Откройте `.htaccess` в редакторе и замените заглушки на ваши актуальные пути:
   - Узнайте ваш путь на хостинге командой `pwd` в терминале BeGet
   - Замените `/home/USERNAME/PATH_TO_PROJECT` на ваш путь
   - Проверьте версию Python (обычно 3.10, 3.8 или другая)

   Пример замены:
   ```apache
   # Было (шаблон):
   PassengerAppRoot /home/USERNAME/PATH_TO_PROJECT
   PassengerPython /home/USERNAME/PATH_TO_PROJECT/venv/bin/python3.10
   
   # Стало (ваш конфиг):
   PassengerAppRoot /home/u/username/my-project/public_html
   PassengerPython /home/u/username/my-project/public_html/venv/bin/python3.10
   ```

### Шаг 2: Создайте файл `passenger_wsgi.py`

1. Скопируйте шаблон:
   ```bash
   cp passenger_wsgi.py.example passenger_wsgi.py
   ```

2. Откройте `passenger_wsgi.py` и проверьте:
   - Версию Python в пути `venv/lib/python3.10/site-packages` (измените на вашу, если отличается)
   - Если автоматическое определение пути не работает, раскомментируйте строку:
     ```python
     # PROJECT_ROOT = '/home/USERNAME/PATH_TO_PROJECT'
     ```
     и впишите ваш полный путь вручную.

### Шаг 3: Создайте файл `.env`

1. Скопируйте шаблон:
   ```bash
   cp .env.example .env
   ```

2. Заполните `.env` вашими секретами (ключи API, пароли БД, SECRET_KEY и т.д.)

### Шаг 4: Установите зависимости

```bash
cd /путь/к/проекту
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 5: Перезапустите приложение

На BeGet для перезапуска Passenger создайте или обновите файл `restart.txt`:

```bash
echo "$(date)" > restart.txt
```

Или через файловый менеджер BeGet измените содержимое файла.

---

## 🔄 Обновление кода на хостинге

После того как файлы настроены, обновление кода происходит безопасно:

```bash
# На хостинге в папке проекта
git pull
```

Файлы `.htaccess`, `passenger_wsgi.py` и `.env` **не будут перезаписаны**, так как они игнорируются Git.

## ✅ Проверка работы

1. Откройте ваш сайт в браузере
2. Проверьте, что приложение загружается без ошибок
3. При проблемах смотрите логи в панели управления BeGet

---

## 🔍 Диагностика проблем

### Ошибка "ModuleNotFoundError"

- Проверьте, что виртуальное окружение активировано и зависимости установлены
- Убедитесь, что путь к `site-packages` в `passenger_wsgi.py` верный

### Ошибка "PassengerAppRoot"

- Проверьте, что путь в `.htaccess` абсолютный и существует
- Убедитесь, что права доступа к папке корректны (755 для директорий, 644 для файлов)

### Приложение не перезапускается

- Убедитесь, что файл `restart.txt` был изменён (новое содержимое или timestamp)
- Проверьте логи Passenger в панели BeGet

```
project-root/
├── .htaccess.example      # Шаблон для .htaccess (в Git)
├── .htaccess              # Рабочий конфиг (НЕ в Git, создаётся вручную)
├── passenger_wsgi.py.example  # Шаблон точки входа (в Git)
├── passenger_wsgi.py      # Рабочий файл (НЕ в Git, создаётся вручную)
├── .env.example           # Шаблон переменных окружения (в Git)
├── .env                   # Рабочие секреты (НЕ в Git)
├── DEPLOY_INSTRUCTION.md  # Эта инструкция (в Git)
└── ...
```

## ⚠️ Важно!

- **Никогда не коммитьте** файлы `.htaccess`, `passenger_wsgi.py` или `.env` с реальными путями
- Используйте только шаблоны `.example` для создания рабочих файлов
- При смене хостинга или переезде просто создайте новые файлы по инструкции
