## 🖥️ Действия на хостинге Beget

После пуша изменений выполните следующие шаги на хостинге:

### Шаг 1: Создайте файл `.htaccess` вручную

Подключитесь к хостингу через FTP или файловый менеджер Beget и создайте файл `.htaccess` в корневой папке вашего проекта с содержимым:

```apache
PassengerEnabled On
PassengerAppRoot /home/u/username/my-project/public_html
PassengerPython /home/u/username/my-project/public_html/venv/bin/python3
```

**⚠️ Важно:** Замените путь `/home/u/username/my-project/public_html` на ваш актуальный путь к проекту на хостинге!

### Шаг 2: Обновите код на хостинге

```bash
# Перейдите в директорию проекта на хостинге
cd /home/u/username/my-project/public_html

# Сделайте pull обновлений
git pull
```

### Шаг 3: Перезапустите приложение

Создайте или обновите файл `restart.txt` в корневой папке проекта:

```bash
# Через SSH
touch restart.txt

# Или просто измените содержимое файла через файловый менеджер
echo "$(date)" > restart.txt
```

## 📄 Пример правильного `.htaccess` для Beget + Passenger

Сохраните этот код в файл `.htaccess` на хостинге (вручную!):

```apache
# Включаем Passenger
PassengerEnabled On

# Указываем корневую директорию приложения (замените на свой путь!)
PassengerAppRoot /home/u/username/my-project/public_html

# Указываем интерпретатор Python из виртуального окружения
PassengerPython /home/u/username/my-project/public_html/venv/bin/python3

# Опционально: устанавливаем переменную окружения
SetEnv FLASK_ENV production

# Опционально: логирование ошибок
PassengerLogDir /home/u/username/my-project/logs
```

## 🔒 Почему это решение надёжно

1. **Независимость от путей**: `passenger_wsgi.py` автоматически определяет свою директорию
2. **Защита конфигов**: `.htaccess` теперь игнорируется git и не перезаписывается при pull
3. **Универсальность**: Код работает на любом хостинге с Passenger, а не только на Beget
4. **Безопасность**: Локальные файлы (.db, логи) не попадают в репозиторий
